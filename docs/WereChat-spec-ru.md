# WereChat — краткая спецификация (ru)

## Назначение

WereChat — это пользовательский почтовый клиент, который работает поверх IMAP/SMTP и отображает часть e‑mail‑переписки в виде чатов (1–1 и в перспективе групповых диалогов).

Ключевые принципы:

- Использует существующие почтовые ящики пользователя, не поднимая собственные серверы и не создавая отдельный мессенджер‑сервис.
- Все сообщения остаются обычными e‑mail‑письмами, совместимыми с классическими клиентаи.
- Интерфейс показывает диалоги как чаты, сохраняя полную совместимость с обычной почтой.
- Поддерживается опциональное шифрование (ГОСТ/«Кузнечик» и OpenPGP) для сообщений между клиентами WereChat.

WereChat не является социальной сетью, не предоставляет публичных каналов и не хранит контент на своих серверах — данные лежат только на почтовых серверах и устройствах пользователя.

## Идентичность и контакты

- Основной идентификатор пользователя — e‑mail‑адрес (например, `user@example.org`).
- Локальное отображаемое имя (`display_name`) используется только в интерфейсе.

Контакты хранятся в локальном файле (например, `contacts.json`) и включают:

- e‑mail адрес и имя контакта.
- схему защиты (`sec_scheme`: `none` | `gost` | `pgp`).
- уровень защиты (`sec_level`: `none` | `json-only` | `full`).
- при `gost` — 32‑байтовый ключ в hex; при `pgp` — публичный ключ, отпечаток и режим (подпись/шифрование/оба).

Для одного сообщения выбирается ровно одна схема защиты; комбинирование нескольких схем для одного письма не допускается.

## Формат сообщений WereChat

Сообщение считается «волкочатовским», если:

- в заголовках письма присутствует `X-WolfChat: 1`;
- клиент может распознать и обработать внутреннее содержимое по спецификации.

Дополнительные заголовки:

```text
X-WolfChat: 1
X-WolfChat-Scheme: none | gost | pgp
X-WolfChat-Level: none | json-only | full
```

Внутреннее содержимое сообщения описывается JSON‑структурой:

```json
{
  "v": 1,
  "type": "text",
  "msg_id": "uuid-строка",
  "from": "you@example.org",
  "to": "alice@example.org",
  "ts": 1741861800,
  "room": null,
  "text": "Привет, это тест",
  "sec": {
    "scheme": "gost",
    "level": "json-only"
  },
  "attachments": [
    {
      "id": "uuid-attachment",
      "kind": "image",
      "mime": "image/jpeg",
      "name": "photo.jpg",
      "size": 123456,
      "encoding": "inline",
      "ref": "cid:photo@wolfchat",
      "duration_ms": 0
    }
  ]
}
```

Основные поля:

- `type` — тип сообщения (text/media/voice/system/…);
- `msg_id` — UUID сообщения;
- `from` / `to` — адреса отправителя и получателя;
- `ts` — UNIX‑время;
- `room` — `null` для 1–1 или идентификатор комнаты для групп;
- `sec` — схема и уровень защиты, согласованные с заголовками письма;
- `attachments` — список вложений с типом, MIME, размером и ссылкой на MIME‑части.

Полная, детальная спецификация (включая варианты MIME‑структур и дополнительные сценарии) описана в основном документе ТЗ и может развиваться по мере развития проекта.
